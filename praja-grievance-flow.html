Proposed System

WhatsApp Bot Web Portal Voice to Text

AI Classification SLA Timer Starts SMS Confirmation
Routed to ward of the citizen

🏛️ CPGRAMS
Centralised Public Grievance Redress & Monitoring System
Current System (Govt of India)
STEP 1
🧑‍🌾
Citizen Files on Portal
Must register on pgportal.gov.in.
English/Hindi only (limited regional language).
No WhatsApp, no voice note support.
Web Portal Only No WhatsApp No Voice Input
Manual routing — 1 to 3 days
STEP 2
📋
Ministry / Dept Nodal Officer
Complaint sent to concerned ministry's nodal officer.
Citizen must correctly identify the ministry.
No AI classification — manual triage only
Manual Routing No AI Priority
Forwarded within ministry hierarchy
STEP 3
🗂️
Department / Field Officer
Assigned to a field-level officer in the dept.
No standard SLA enforced at local level.
Officer can mark "under process" indefinitely.
No Enforced SLA Avg. 30–60 days
Sits here for weeks...
⚠️ PROBLEM
🚫
Complaint Gets Stuck
No automatic escalation exists in CPGRAMS.
Citizen has no visibility into where it is.
Common reply: "Under examination" — for months.
No ward-level elected representative involvement.
No Auto-Escalation No Local Leader Opaque Process
Citizen must manually file a reminder
STEP 4
📝
Citizen Files a Reminder / Appeal
Citizen must log in again and file a reminder.
Goes back to the same nodal officer.
No political accountability, no sentiment tracking.
Manual Reminder No Accountability
Eventually — if lucky
OUTCOME
📁
Marked Closed (Often Prematurely)

No photo proof, no citizen rating.
No audit trail of who handled what.
Avg. resolution: 30–90+ days
No Citizen Rating No Proof Required 30–90+ days
