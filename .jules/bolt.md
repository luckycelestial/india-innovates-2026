## 2026-04-26 - Optimized Date parsing in React Component Data Processing
**Learning:** Instantiating `new Date(dateString)` in frequent loops (e.g., inside `map`, `filter`, or `forEach` when iterating through list of records like grievances) introduces heavy memory allocation and garbage collection overhead. Since we mostly just need to evaluate and compare against timestamps (`getTime()`), creating full Javascript Date objects is wasteful.
**Action:** Always favor `Date.parse(dateString)` to extract raw millisecond timestamps directly from valid string dates when doing comparisons or arithmetic in arrays/loops, saving both CPU cycles and memory. Additionally, extract evaluation objects like `new Date()` out of `filter`/`map` callbacks when they evaluate to a static constant value like current time `Date.now()`.
